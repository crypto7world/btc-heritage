# Wallets sometime need to be restored, there is 2 fundamentaly different part to restore
# - the seed; for all things related to private keys
# - the descriptors; for all things related to TapRoot/Heritage configurations
# It will happen that only one of the two parts need restoration
# Furthermore, it's different for the service or for a local wallet

# The procedure will always to first ensure the "public" side is restored
heritage-cli service-wallet [--service-url <service_URL>] <wallet-name> restore <JSON-backup-file> [--seed <mnemonic> [--with-password] | --ledger]
heritage-cli local-wallet <wallet-name> restore <JSON-backup-file> [--seed <mnemonic> [--with-password] | --ledger]

# If the service-wallet already exist
